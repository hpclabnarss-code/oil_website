# Oil Spill Shapefile Backend

A small FastAPI backend that stores your oil spill shapefiles in a real
database and serves them to the map viewer page, plus an admin API for
uploading new shapefiles.

## What's inside

```
oilspill-backend/
  app/
    main.py          FastAPI app + all routes
    models.py         SpillRecord database table
    database.py        SQLite engine/session setup
    schemas.py         Request/response validation
    security.py         Admin login + token check
    geo_utils.py         Shapefile zip parsing, reprojection, area calc
  uploads/              Original uploaded .zip shapefiles are stored here
  seed_data.py          Optional: loads 15 demo records to test with
  requirements.txt
```

Storage: SQLite by default (a single `oilspill.db` file, zero setup). Geometry
is stored as GeoJSON text, so swapping to Postgres later (set the
`DATABASE_URL` env var) needs no code changes, and you can add a `PostGIS`
geometry column on top if you outgrow this.

Shapefile parsing uses **pyshp** + **pyproj** only ŌĆö no GDAL system install
required. It reprojects automatically if the shapefile's `.prj` is not
already WGS84 (lon/lat), and computes an approximate geodesic area in km┬▓.

## 1. Install & run

```bash
cd oilspill-backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

The API is now at `http://localhost:8000`. Interactive API docs (auto
generated by FastAPI) are at `http://localhost:8000/docs`.

## 2. Set your admin password

```bash
export ADMIN_USERNAME="admin"
export ADMIN_PASSWORD="choose-a-strong-password"
```

Set these *before* starting uvicorn. If you skip this, the default is
`admin` / `changeme123` ŌĆö fine to test with, but change it before this is
reachable from outside your machine.

## 3. (Optional) load demo data

```bash
python seed_data.py
```

Loads 15 sample spill records so the viewer page has something to show
immediately. Skip this once you're uploading your real shapefiles.

## 4. Point the frontend pages at this server

Open `oil-spill-viewer.html` and `admin-upload.html` in a text editor and
set, near the top of the `<script>` block:

```js
const API_BASE = "http://localhost:8000/api";
```

to wherever you're actually running this (e.g. `http://your-server-ip:8000/api`).

## Uploading a shapefile (what the admin page sends)

The admin page zips up your `.shp` + `.shx` + `.dbf` (+ `.prj` if you have
one) and POSTs to `/api/admin/upload` with:

| field        | required | notes                                   |
|--------------|----------|------------------------------------------|
| `file`       | yes      | the `.zip` containing the shapefile      |
| `name`       | yes      | display name for the spill               |
| `spill_date` | yes      | `YYYY-MM-DD`                              |
| `severity`   | yes      | `critical` / `high` / `medium` / `low`   |
| `region`     | no       | for your own organization/records        |
| `source`     | no       | e.g. "Tanker accident"                    |
| `vessel`     | no       |                                            |
| `oil_type`   | no       |                                            |
| `status`     | no       | defaults to "Active"                      |
| `spill_id`   | no       | custom ID; auto-generated if omitted      |

The backend extracts the geometry, reprojects to WGS84 if needed,
computes the area, and stores everything as one record. The original zip
stays on disk under `uploads/` and can be re-downloaded from the admin
screen.

## Moving this to a real server

- Run behind a reverse proxy (nginx/Caddy) with HTTPS.
- Restrict `allow_origins` in `app/main.py`'s CORS middleware to your
  actual frontend domain instead of `"*"`.
- Put `uvicorn` behind a process manager (systemd, supervisor, or
  `pm2`) so it restarts on crash/reboot, or run with
  `uvicorn app.main:app --workers 4` behind gunicorn for more throughput.
- Consider swapping SQLite for Postgres if multiple people will be
  uploading concurrently.
